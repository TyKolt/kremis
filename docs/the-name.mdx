**Kremis** is a coined word. It has no etymology, no acronym, no hidden meaning.

## Pronunciation

Two syllables, stress on the first.

| Form | Value |
|------|-------|
| IPA | `/ˈkrɛ.mɪs/` |
| Plain English | **KREH-miss** |
| Stress | First syllable |
| Letters | K · R · E · M · I · S |

Common mispronunciations to avoid:

- **"KREEM-iss"** — the first vowel is short (`/ɛ/`, as in *bed*), not long.
- **"kreh-MISS"** — the stress falls on the first syllable, never on the second.

In Italian, German, French, and Spanish the spelling reads phonetically: same stress, same vowels.

## Origin

Kremis is a coined name. It does not derive from Latin, Greek, English, or any other language. There is no backronym. There is no metaphor.

The choice is deliberate.

Most projects in the knowledge-graph and AI infrastructure space pick names that gesture at their function: *LangChain* chains language, *LlamaIndex* indexes for LLaMAs, *Weaviate* weaves vectors, *Pinecone* evokes a tree-like structure. The pattern is clear — the name is a metaphor for what the project does.

Kremis takes the opposite path. A deterministic substrate does not need a metaphor to explain itself; it explains itself by being inspectable. Every node traces back to a signal. Every result reproduces. The name only has to do one job: identify the project unambiguously — in a search bar, in a CLI, in a commit message, in any locale.

A coined word does that better than a borrowed one.

## What the Name Is For

A name is a handle, not a manifesto. Kremis means nothing on its own. The project defines the word, not the other way around.

For what Kremis *does*, read the [Design Philosophy](/philosophy). For what Kremis *is*, the substrate is the answer.
